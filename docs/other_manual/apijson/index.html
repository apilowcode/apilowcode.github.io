APIJSON 语法

apijson项目文档地址

新增​

JSON模板：

{
  "Moment": {
    "content": "今天天气不错，到处都是提拉米苏雪",
    "userId": 38710
  }
}

响应结果：

{
  "Moment": {
    "code": 200,
    "count": 1,
    "id": 1544520921923,
    "msg": "success"
  },
  "code": 200,
  "msg": "success"
}

返回的 id 是新增数据的主键值。

修改​

JSON模板：

{
  "Moment": {
    "id": 1544520921923,
    "content": "海洋动物数量减少，如果非吃不可，不点杀也是在保护它们"
  }
}

响应结果：

{
  "Moment": {
    "code": 200,
    "count": 1,
    "id": 1544520921923,
    "msg": "success"
  },
  "code": 200,
  "msg": "success"
}

删除​

JSON模板：

{
  "Moment": {
    "id": 1544520921923
  }
}

查询​

字段过滤​

使用 @column 筛选需要的字段，多个字段用逗号隔开：

JSON模板：

{
  "Moment": {
    "id": 12,
    "@column": "content"
  }
}

响应结果：

{
  "Moment": {
    "content": "1111534034"
  },
  "code": 200,
  "msg": "success"
}

字段别名​

"@column":"id,date:time,content:text" 中 date:time 表示将 date 字段重命名为 time：

{
  "Moment": {
    "id": 12,
    "@column": "id,date:time,content:text"
  }
}

响应结果：

{
  "Moment": {
    "text": "1111534034",
    "time": "2017-02-08 16:06:11.0",
    "id": 12
  }
}

逻辑运算-筛选​

组合多个条件筛选符合要求的数据。

筛选 id 在 12、15、32 中的数据：

JSON模板：

{
  "[]": {
    "Moment": {
      "id{}": [12, 15, 32],
      "@column": "id,date,content"
    }
  }
}

响应结果：

{
  "[]": [
    {
      "Moment": {
        "content": "1111534034",
        "date": "2017-02-08 16:06:11.0",
        "id": 12
      }
    },
    {
      "Moment": {
        "content": "APIJSON is a JSON Transmission Structure Protocol…",
        "date": "2017-02-08 16:06:11.0",
        "id": 15
      }
    },
    { "Moment": { "date": "2017-02-08 16:06:11.0", "id": 32 } }
  ],
  "code": 200,
  "msg": "success"
}

逻辑运算符​

符号含义示例
&AND"id&{}":">=10,<=40"
|OR（默认）"id|{}":"<=300,>=400" 等价于 "id{}":"<=300,>=400"
!NOT"id!{}":[12,15,32] 表示 id 不在这些值内

范围查询："id&{}":">=300,<=400" 等价于 SQL (id >= 300 AND id <= 400)

多条件组合：

{
  "[]": {
    "Moment": {
      "id&{}": ">=10,<=40",
      "id!{}": [12],
      "@column": "id,date,content:text"
    }
  }
}

模糊查询​

支持 $（LIKE）和 ~（包含）两种模糊查询方式：

{
  "[]": {
    "Moment": {
      "content$": "%APIJSON%",
      "@column": "id,date,content:text"
    }
  }
}
模式说明
keyword%以 keyword 开头
%keyword以 keyword 结束
%keyword%包含 keyword 的字符串
%k%e%y%包含字母 k, e, y

"content~":"keyword" 等价于 "content$":"%keyword%"，表示包含某字符串。

正则匹配​

{
  "[]": {
    "Moment": {
      "content~": "^[0-9]+$",
      "@column": "id,date,content:text"
    }
  }
}

正则表达式 ^[0-9]+$ 查询 content 为纯数字的数据。

列表数据​

MySQL 中 JSON 数组类型的字段，可用 <> 运算符查询包含特定元素的记录：

{
  "[]": {
    "Moment": {
      "praiseUserIdList<>": 82001,
      "@column": "id,date,content,praiseUserIdList"
    }
  }
}

响应结果：

{
  "[]": [
    {
      "Moment": {
        "date": "2017-02-08 16:06:11.0",
        "id": 32,
        "praiseUserIdList": [38710, 82002, 82001]
      }
    },
    {
      "Moment": {
        "content": "This is a Content...-435",
        "date": "2017-02-01 19:14:31.0",
        "id": 58,
        "praiseUserIdList": [38710, 82003, 82001]
      }
    }
  ],
  "code": 200,
  "msg": "success"
}

分页​

使用 page 和 count 实现分页，page 从 0 开始，count 为每页数量：

{
  "[]": {
    "Moment": {
      "@column": "id,date,content,praiseUserIdList"
    },
    "page": 0,
    "count": 5
  }
}

对应 SQL：SELECT ... LIMIT 5 OFFSET 0

查询总数​

通过 query 参数控制：1 只返回总数，2 同时返回数据和总数。

{
  "[]": {
    "Moment": {
      "@column": "id,date,content"
    },
    "query": 2
  },
  "total@": "/[]/total"
}

排序​

使用 @order 指定排序规则，字段后跟 +（升序，默认）或 -（降序），字段前后顺序决定排序优先级：

{
  "[]": {
    "Moment": {
      "@column": "id,date,content",
      "praiseUserIdList<>": 38710,
      "@order": "date-,id+"
    }
  }
}

关联查询​

通过引用路径关联多张表，实现类似 SQL JOIN 的查询：

{
  "[]": {
    "Moment": {
      "@column": "id,date,userId",
      "id": 12
    },
    "User": {
      "id@": "/Moment/userId",
      "@column": "id,name"
    }
  }
}

其中 "id@":"/Moment/userId" 表示 User.id 与 Moment.userId 关联。/Moment 是 [] 下的 Moment 对象，本例等价于完整路径 []/Moment/userId。

响应结果：

{
  "[]": [
    {
      "Moment": { "date": "2017-02-08 16:06:11.0", "id": 12, "userId": 70793 },
      "User": { "id": 70793, "name": "Strong" }
    }
  ],
  "code": 200,
  "msg": "success"
}

分组查询​

使用 @group 进行分组聚合，支持 count、sum、max、min、avg 等 SQL 函数。

单独求最大值：

{
  "Moment": {
    "@column": "max(id):maxid"
  }
}

响应结果：

{
  "Moment": { "maxid": 1541912160047 },
  "code": 200,
  "msg": "success"
}

分组聚合，统计每个店铺的总营业额：

{
  "[]": {
    "Sale": {
      "@column": "store_id;sum(amt):totAmt",
      "@group": "store_id"
    }
  }
}

注意：SQL 函数在 @column 中需要用分号 ; 隔开。
