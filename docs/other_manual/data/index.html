数据格式

定义标准的输入输出数据格式规范，完全兼容 amis 前端库的数据格式规范。
后台根据接口模板与输入参数，将 JSON 模板接口编译成 JSON 后调用 APIJSON 库、SQL 模板接口编译成标准 SQL 后调用 JDBC 库，以此实现数据库的增删改查操作，并根据接口类型将操作结果组装成对应的数据格式返回。

输入参数关键字​

关键字描述默认值生效范围
orderBy排序字段无原始查询、分页查询、下拉查询
orderDir排序方法asc 或 desc原始查询、分页查询、下拉查询
page分页页码1原始查询、分页查询、下拉查询、目录查询
perPage分页大小10原始查询、分页查询、下拉查询、目录查询

输入数据格式​

低代码接口开发平台发布的所有接口同时支持 GET 与 POST 两种模式调用，GET 模式输入参数为 HTTP GET 参数，POST 模式输入参数为 HTTP 请求体。

GET 方法数据格式​

参数附加在 URL 的问号后面

  • 用 ? 分隔 URL 和参数
  • 多个参数用 & 连接
  • 键值对格式：key=value
  • 需要进行 URL 编码以处理特殊字符
http://ip:port/system_code/module_code/resource_code/interface_code?name=张三&age=25&city=北京

POST 方法数据格式​

输入参数为 JSON 对象，填充在请求体中，使用 POST 方法提交。

{
  "param_key_1": "param_value_1",
  "param_key_2": "param_value_2",
  "param_key_3": "param_value_3"
}

返回数据格式​

低代码接口开发平台返回的数据都符合下面的格式：

关键字描述
status0 操作成功，1 操作失败
message操作结果描述
datakey-value 结构的对象，有可能为空对象
{
  "status": 0,
  "message": "",
  "data": {
    ...其他字段
  }
}

新增/编辑/删除/操作​

关键字描述
status0 操作成功，1 操作失败
message操作结果描述
data.id操作对象的ID
{
    "status": 0,
    "message": "success",
    "data":{
        "id": ""
    }
}

单条查询​

关键字描述
status0 操作成功，1 操作失败
message操作结果描述
dataJSON 格式查询结果，key 表示字段名称，value 表示字段值

分页查询​

关键字描述
status0 操作成功，1 操作失败
message操作结果描述
data.count满足查询条件的总记录条数
data.rows满足查询条件的记录集

下拉查询​

关键字描述
status0 操作成功，1 操作失败
message操作结果描述
data满足查询条件的总记录集
data[].value下拉组件对应组件值，一般将 ID 字段重命名成 value 字段
data[].label下拉组件对应显示值
data[].其他字段扩展字段，根据查询模板附加，用于显示或者下拉组件事件执行事件联动赋值

目录树查询​

关键字描述
status0 操作成功，1 操作失败
message操作结果描述
data[].value目录树ID
data[].label目录树名字
data[].parent_id目录树父ID
data[].weight目录树排序字段
data[].children目录树子节点数组，无子节点则无该字段

原始查询​

原始查询直接返回原始查询数据，可配合后处理代码自定义数据返回格式。
