导出导入

日常开发在开发环境中完成，接口开发完毕后，从开发环境导出已通过测试的接口配置文件，导入生产环境，实现开发与生产环境解耦。导入后，请根据生产环境的实际情况修改数据库连接配置。除非已验证 SQL 模板兼容不同类型的数据库，否则不建议更改数据库类型。

导出​

导入​

导入前请务必先执行导出操作，以备份当前数据库及接口配置
点击导入按钮，弹出对话框，选择 JSON 格式的接口配置文件，点击确认，执行导入操作。

注意事项​

  • 数据库导入策略：以数据库 ID 为主键，仅写入尚不存在的记录，已存在的记录将跳过。
  • 接口导入策略：以接口 URI 编号作为唯一值判断依据，对不存在的记录执行写入，对已存在的记录进行更新。
  • 导入完成后请重启服务，以确保接口配置即时生效。
