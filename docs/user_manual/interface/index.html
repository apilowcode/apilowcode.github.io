接口开发

接口设计、开发、测试管理

输入参数关键字

关键字描述默认值生效范围
orderBy排序字段无原始查询、分页查询、下拉查询
orderDir排序方法asc 或 desc原始查询、分页查询、下拉查询
page分页页码1原始查询、分页查询、下拉查询、目录查询
perPage分页大小10原始查询、分页查询、下拉查询、目录查询

接口列表​

展示现有接口集合，默认按修改时间倒序排列。左侧边栏支持通过接口目录进行快速筛选；同时支持基于接口 URI、名称、类型及状态的多维度过滤。

批量新增​

点击「批量新增」按钮，系统将弹出配置表单。您只需依次填写系统、模块、资源归属，完成数据源（数据库、库、表）及表字段等相关配置，核对无误后点击「保存」，即可一次性批量生成接口。 后台将基于标准 JSON 模板，自动为单表创建新增、编辑、删除、分页查询及单条查询共 5 个标准化接口，将重复性编码工作转化为简单配置，显著降低手工逐个建接口的繁琐与出错风险。

新增JSON接口​

使用 JSON 模板开发接口，兼容 APIJSON 语法。支持以 JSON 格式对数据库对象进行新增、编辑、删除和查询操作。尤其在新增、编辑与删除场景中，相比 SQL 模板更为简洁。通过配置 JSON 模板即可实现接口功能，能够大幅简化接口开发工作，在减少80%后端代码量的同时，保障接口的实现质量。

新增|编辑|删除接口​

通过 JSON 模板添加新增、编辑、删除操作的接口。选择数据库表和接口操作类型后，系统会自动生成对应 JSON 格式模板。新增、编辑、删除操作所使用的 JSON 模板完全一致，后台会根据操作类型调用 APIJSON 库中的不同方法。

基本信息​

JSON 模板​

关键字描述
DATA_TYPE_TABLE表示待操作表名且表名首字母必须大写
"&": "$$"表示填充请求参数中除使用 ${} 标记字段之外的所有剩余参数
"id": "${id}"表示从请求参数的 JSON 中获取 id 节点的参数值并填充到对应节点
"@schema": "crud"表示schema为crud，翻译成SQL就是表全名 crud.DATA_TYPE_TABLE
{
  "DATA_TYPE_TABLE": {
    "&": "$$",
    "id": "${id}",
    "@schema": "crud"
  }
}

单条查询接口​

通过 JSON 模板新增按主键 id 查询单条记录的操作接口。选择数据库表和接口操作类型后，系统将自动生成对应的 JSON 格式模板。

JSON 模板​

关键字描述
DATA_TYPE_TABLE表示待操作表名且表名首字母必须大写
"id": "${id}"表示从请求参数的 JSON 中获取 id 节点的参数值并填充到对应节点
"@schema": "crud"表示schema为crud，翻译成SQL就是表全名 crud.DATA_TYPE_TABLE
{
  "DATA_TYPE_TABLE": {
    "id": "${id}",
    "@schema": "crud"
  }
}

分页查询接口​

通过 JSON 模板实现分页查询。选择数据库表及操作类型后，系统将自动生成对应的 JSON 请求模板。排序与分页字段需由用户在接口请求参数中手动填充。

JSON 模板​

关键字描述
"[]"在 APIJSON 语法中表示查询集合
DATA_TYPE_TABLE表示待操作表名且表名首字母必须大写
"&": "$$"表示填充请求参数中除使用 ${} 标记字段与参数关键字之外的所有剩余参数
"id": "${id}"表示从请求参数的 JSON 中获取 id 节点的参数值并填充到对应节点
"@schema": "crud"表示schema为crud，翻译成SQL就是表全名 crud.DATA_TYPE_TABLE
{
  "[]": {
    "DATA_TYPE_TABLE": {
      "&": "$$",
      "@schema": "crud"
    },
    "query": 2
  },
  "total@": "/[]/total"
}

下拉查询接口​

适配前端下拉组件，返回对应格式数据，支持数据排序与分页。

JSON 模板​

节点描述
[]在 APIJSON 语法中表示查询集合
DATA_TYPE_TABLE表示待操作表名且表名首字母必须大写
"&": "$$"表示填充请求参数中除使用 ${} 标记字段与参数关键字之外的所有剩余参数
@column查询字段，必须包含 label 与 value 字段，label 是下拉栏显示的值，value 是组件的实际值，column_1:label 表示 column_1 字段重命名为 label，多字段用英文逗号隔开
@group重复记录使用分组去重，字段名为重命名前的字段名，多字段用英文逗号隔开
@schema表示schema为crud，翻译成SQL就是表全名 crud.DATA_TYPE_TABLE
{
  "[]": {
    "DATA_TYPE_TABLE": {
      "&": "$$",
      "@column": "column_1:label,column_2:value,column_3,column_4",
      "@group": "column_1,column_2",
      "@schema": "crud"
    },
    "query": 2
  },
  "total@": "/[]/total"
}

目录树查询接口​

适配前端目录树组件，返回对应格式数据。

JSON 模板​

节点描述
[]在 APIJSON 语法中表示查询集合
ORGAN表示待操作表名且表名首字母必须大写
"&": "$$"表示填充请求参数中除使用 ${} 标记字段与参数关键字之外的所有剩余参数
@column查询字段，必须包含 value、label、parent_id、weight 字段，分别表示 目录 ID、目录名称、父目录 ID、目录排序依据
@schema表示schema为crud，翻译成SQL就是表全名 crud.DATA_TYPE_TABLE
{
  "[]": {
    "ORGAN": {
      "&": "$$",
      "@column": "id:value,parent_id:parent_id,organ_name:label,organ_code:weight",
      "@schema": "crud"
    },
    "query": 2
  },
  "total@": "/[]/total"
}

原始查询接口​

使用 APIJSON 执行多表关联查询，建议优先采用 SQL 模板处理复杂逻辑，从而降低开发者的学习曲线。如确需使用框架原生语法，请严格参照 APIJSON 标准规范执行。 JSON 模板的参数填充语法与其他接口操作类型保持一致。

新增 SQL 接口​

使用 SQL 模板开发接口，兼容 MyBatis 语法。支持以 SQL 格式对数据库对象进行新增、编辑、删除和查询操作。尤其在复杂查询场景中，相比 JSON 模板更容易理解，减少学习成本与上手难度。通过配置 SQL 模板即可实现接口功能，能够大幅简化接口开发工作，在减少80%后端代码量的同时，保障接口的实现质量。

新增接口​

通过 SQL 模板创建新增接口。在接口管理界面中，点击"新增 SQL 接口"按钮，弹出新增 SQL 接口表单，选定数据库表，操作类型选新增，系统将自动生成符合 MyBatis 语法的 SQL 插入模板。

基本信息​

SQL 模板​

后台自动根据表结构生成符合 MyBatis 语法的 SQL 新增语句模板，开发人员可根据实际业务调整。 系统约定所有前后端交互过程中传递的参数名称均使用蛇形命名(下划线 _ 分隔单词、全部小写的编程命名规范)

insert into crud.organ( id,
 <if test="organ_code != null"> organ_code,</if>
 <if test="organ_name != null"> organ_name,</if>
 <if test="parent_id != null"> parent_id,</if>)
 values
( #{id},
 <if test="organ_code != null"> #{organ_code},</if>
 <if test="organ_name != null"> #{organ_name},</if>
 <if test="parent_id != null"> #{parent_id},</if>)

删除接口​

通过 SQL 模板创建删除接口。在接口管理界面中，点击"新增 SQL 接口"按钮，弹出新增 SQL 接口表单，选定数据库表，操作类型选删除，系统将自动生成符合 MyBatis 语法的 SQL 删除模板。

基本信息​

SQL 模板​

后台自动根据表结构生成符合 MyBatis 语法的删除 SQL语句模板。

delete from crud.organ where id=#{id}

编辑接口​

通过 SQL 模板创建编辑接口。在接口管理界面中，点击"新增 SQL 接口"按钮，弹出新增 SQL 接口表单，选定数据库表，操作类型选编辑，系统将自动生成符合 MyBatis 语法的 SQL 更新模板。

基本信息​

SQL 模板​

后台自动根据表结构生成符合 MyBatis 语法的 SQL 更新语句模板。

update  crud.organ
<set>
<if test="organ_code != null"> organ_code=#{organ_code},</if>
<if test="organ_name != null"> organ_name=#{organ_name},</if>
<if test="parent_id != null"> parent_id=#{parent_id},</if>
</set>
 where  id = #{id}

单条查询接口​

通过 SQL 模板创建单条查询接口。在接口管理界面中，点击"新增 SQL 接口"按钮，弹出新增 SQL 接口表单，选定数据库表，操作类型选单条查询，系统将自动生成符合 MyBatis 语法的 SQL 查询模板。

基本信息​

SQL 模板​

后台自动根据表结构生成符合 MyBatis 语法的 SQL 查询语句模板，如果匹配到多条只取第一条。

select * from crud.organ where id=#{id}

分页查询接口​

通过 SQL 模板创建分页查询接口。在接口管理界面中，点击"新增 SQL 接口"按钮，弹出新增 SQL 接口表单，选定数据库表，操作类型选分页查询，系统将自动生成符合 MyBatis 语法的 SQL 查询模板，单次最大支持返回 1000 条数据。

基本信息​

SQL 模板​

后台自动根据表结构生成符合 MyBatis 语法的 SQL 查询语句模板，自动适配分页与排序参数，自动识别字段模糊查询配置。

select *
 from crud.organ
 where
<if test="id != null"> id=#{id} and</if>
<if test="organ_code != null"> organ_code=#{organ_code} and</if>
<if test="organ_name != null"> organ_name=#{organ_name} and</if>
<if test="parent_id != null"> parent_id=#{parent_id} and</if>
 1=1

下拉查询接口​

通过 SQL 模板创建下拉查询接口。适用于下拉组件数据绑定，在接口管理界面中，点击"新增 SQL 接口"按钮，弹出新增 SQL 接口表单，选定数据库表，操作类型选下拉查询，系统将自动生成符合 MyBatis 语法的 SQL 查询模板，单次最大支持返回 1000 条数据。

基本信息​

SQL 模板​

后台自动根据表结构生成符合 MyBatis 语法的 SQL 查询语句模板，自动适配分页与排序参数，自动识别字段模糊查询配置。

字段描述
id as value值字段必须命名成 value，是下拉组件的实际值
organ_name as label标签字段必须命名成 label，是下拉组件的显示值
field_2,field_3可以添加其他字段，与 value、label 一并返回，用于附带扩展信息
select
id as value,organ_name as label,field_2,field_3
from crud.organ_name

目录树查询接口​

通过 SQL 模板创建目录树查询接口。适用于目录组件数据绑定，在接口管理界面中，点击"新增 SQL 接口"按钮，弹出新增 SQL 接口表单，选定数据库表，操作类型选目录查询，系统将自动生成符合 MyBatis 语法的 SQL 查询模板，单次最大支持返回 1000 条数据。

基本信息​

SQL 模板​

后台自动根据表结构生成符合 MyBatis 语法的 SQL 查询语句模板，自动适配分页参数，自动识别字段模糊查询配置，SQL 模板根据实际表结构调整。

字段描述
id as value值字段必须命名成 value，是树组件的实际值
organ_name as label标签字段必须命名成 label，是树组件的显示值
parent_id父级 ID 字段必须命名成 parent_id，树查询必须指定
organ_code as weight权重字段必须命名成 weight，用于树组件节点排序
select
id as value,parent_id,organ_name as label,organ_code as weight
from schema.table

原始查询接口​

通过 SQL 模板创建查询接口，直接返回原始查询数据，可配合后处理代码自定义数据返回格式。

编辑接口​

在接口开发界面中，点击操作栏的"编辑"按钮，弹出编辑表单。

  • 编辑参数：修改接口编号、接口名称，字段配置、代码配置、JSON/SQL 模板，参数校验。
  • 测试接口：测试接口输入输出。
  • 代码测试：测试预处理与后处理代码。

测试接口​

新增接口保存成功后，系统会默认将其排列在列表第一位，方便您快速定位。 点击接口操作栏中的“接口测试”按钮，立即对接口进行调试，并通过查看响应结果来验证输入输出数据的正确性。
详细输入参数及返回数据格式请在接口测试页面中查看响应示例。

停用接口​

在接口开发界面中，点击操作栏的"停用"按钮，将接口设置为停用状态。

  • 开发模式：停用状态下的接口无法通过接口地址从外部调用，会返回接口未注册提示。
  • 生产模式：停用接口需要重启服务。
  • 允许删除：停用状态下的接口允许执行删除操作。

删除接口​

在接口开发界面中，点击操作栏的"删除"按钮，删除接口配置，删除之前请导出接口配置，以防误操作。
